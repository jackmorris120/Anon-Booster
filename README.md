# Cooking Effective Tests with Ingredients from Developer Tests

## Summary

This repository contains the implementation of **Booster** and its experimental results for the paper "Cooking Effective Tests with Ingredients from Developer Tests"

Booster is an intelligent test case generation tool that amplifies developer written tests by reusing and assembling existing inputs from an initial test suite. Unlike existing approaches that focus on coverage improvement or oracle enhancement, Booster achieves bug detection enhancement without requiring additional resources for coverage or oracle improvement. Booster generates new test cases using only existing inputs from the initial test suite no new inputs are created.

To evaluate Booster's effectiveness, we apply it to initial test suites from Defects4j benchmarks 17 different projects. We then measure bug detection and coverage improvements and compare the results against the existing tools.

This repository provides:
- Complete source code and executable JAR of Booster
- Experimental results in CSV format
- Detailed usage instructions

## Key Features

- **Sequence Chaining (seqc)**: Automatically identifies required objects and arranges code sequences for object creation
- **Sequence Preserving (seqp)**: Masks primitive and literal values, then assigns new values of matching types
- **Regression and Error-Revealing Testing**: Generates test oracles to detect both behavioral changes and error conditions
- **Incremental File Output**: Splits generated test cases across multiple files

## Requirements

- **Java**: Java 8 or higher
- **Gradle**: 7.0 or higher
- **Libraries**: JUnit 4.12+, Spoon 9.0+, EvoSuite 1.1.0+, Apache Commons Lang 3.0+, Apache Commons IO 2.0+
- **Memory**: Minimum 8GB RAM (32GB+ recommended)

## Experimental Results

All experimental results are located in the `Data/` directory:

### Bug Detection Results (`Data/Bug Detection/`)
Comparison of bug detection capability across:
- **Booster**: Results from applying Booster to existing developer written test suites
- **EvoSuite**: Tests generated by EvoSuite
- **EvoAmp**: Tests generated by EvoAmp (EvoSuite variant)
- **Randoop**: Tests generated by Randoop
- **DSpot**: Tests generated by DSpot

Results include both regression testing and error-revealing testing modes.

### Coverage Analysis (`Data/Coverage/`)
Code coverage results for each tool's test suites on Defects4J bugs.

### Ablation Study (`Data/Ablation/`)
Performance comparison of Booster with various components disabled:
- Booster without Combinatorial Mutation
- Booster without Seed Augmentation
- Booster with Different Assembling Methods
